1. Difference between map(), filter(), forEach() and find() function

map() function applies a function over an original array to do the operation of that function by iterating each element of the original array. Finally, it returns a new array as the original array with transformed elements.

filter() function works over an array based on a condition. If the array elements passed the condition then filter() function filtered and returns that elements with creating a new array.

forEach() function can't store in a variable because it returns undefined and doesn't create a new array like map() and filter() function. It iterates each element of an array and executes a function for each element.

find() function find an element of the array according to the condition but only the first element of that array if the condition is satisfied .

2. Difference between Local Storage, Session Storage and Cookies

Local Storage is a storage of browser where key and value pairs of data stored in the browser. This data will remain until cleaned. That means no expiration time for the local storage data. Even PC is restarted and browser is closed, the data will still found in the local storage.

Session Storage is almost similar to the local storage. But the data has expiration time. If browser tab is closed, session storage data will remove automatically. It's a temporary storage of the browser.

Cookies are used for storing user preferences, login information, and other session-related data. cookies can be set with an expiration time, after which they get automatically deleted. Cookies can also be set with different security and privacy settings, such as HttpOnly, Secure, SameSite, etc.

3. What is Event Loop and how does Event Loop works?

The event loop is a process that waits for the Call Stack to be clear before pushing callbacks from the Task Queue to the Call Stack. Once the Stack is clear, the event loop triggers and checks the Task Queue for available callbacks. If there are any, it pushes it to the Call Stack, waits for the Call Stack to be clear again, and repeats the same process.
